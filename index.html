Introduction

Ignition is an app designed to simulate a test drive with certain parameters to be enabled or disable to simulate conditions.

Design

Designed in FIGMA and implemented in Android Studio layout. Design is ambiental and relatively simplistic

App features

The following major functions are identified:

  • Select destination to simulate drive
  • Increase/Decrease speed
  • Change configuration of the parameters in real-time affecting fuel consumption
  • Display report on phone screen
  • Export report of data to PDF
  • Share report to social media

QOC - Questions, Options, Criteria

  • How the user is introduced to the application?

    • At start the user is presented with a splash screen for application introduction.

    There were was an alternative to open the app either on a login interface or directly in the functionality, neither were pleasant

    This approach offers the user the most pleasant, profi experience.

  • How we first interact with the user?

    • The functionalities are generated with the use of images which gives the user an intrigue to experiment.

    An alternative would've been to spell everything outright but that is a poor approach to UI design.

    Image use as buttons for most of the activities are a design choice with the goal of making the UI as pleasant as possible.

  • What comes first?

    • At first, the user needs to input the destination to start the test drive, afterwards parameters can be changed in real time

    Another approach would've been to input all parameters first, not just the destination but since the parameters can be changed live, there was no good reason for that choice to be made

    Requiring very little of the user to start the functionality makes the app easier to use
